# UI Regression Analysis (Post `REVIEW_FINDINGS.md` Fixes)

## Scope

This report identifies the changes made in the three `REVIEW_FINDINGS.md`-driven commits that could cause the observed UI regression (landing/components/docs/templates partially missing or not rendering as before).

Analyzed commits:

1. `b0304b7` - `fix security hardening in api and cli flows`
2. `4911e2c` - `update dynamic and particle components across registry sync`
3. `36ec636` - `docs update review findings progress tracker`

---

## High-confidence Root Cause

### 1) CSP change in `apps/www/next.config.ts` (commit `b0304b7`) - **Very likely**

Change made:

- `script-src` changed from:
  - `script-src 'self' 'unsafe-inline'`
- to:
  - `script-src 'self' 'sha256-xCfqQI16dMbRG5JxRw/siqAUJy12Eg5vEBtZJiUS0fA='`

Why this is risky:

- Next.js pages rely on runtime/hydration scripts.
- A single fixed hash typically does not cover all inline/runtime scripts generated by Next across routes/builds/dev.
- When blocked by CSP, pages can render only partial static HTML, with missing interactive/loaded sections (matching the reported symptom: routes exist, UI sections missing).

Observed correlation:

- The regression appeared after this commit sequence.
- Restoring UI/app files to commit `7cd5544...` (which uses the older CSP setting) restored expected UI behavior.

---

## Secondary Contributors (Possible but lower confidence)

### 2) Large synced component updates in `4911e2c` - **Possible**

Files changed include:

- `apps/www/components/ui/dynamic-info.tsx`
- `apps/www/components/ui/dot-grid-background.tsx`
- `apps/www/components/ui/particle-field.tsx`
- mirrored registry component files
- regenerated registry manifests/json artifacts

Risk profile:

- These are substantial runtime component refactors and generated content syncs.
- Any runtime exception in these components could affect pages where they render.
- However, this is less likely to explain broad multi-route section disappearance by itself compared to CSP script blocking.

Notes:

- Registry entry names/paths stayed intact; changed payload was mainly `files[0].content` for the affected components.
- This points to behavior/content changes, not registry index structural deletion.

---

## Not a UI Cause

### 3) `36ec636` - `REVIEW_FINDINGS.md` only - **Not causal**

- Changed file: `REVIEW_FINDINGS.md` only.
- No app/runtime/UI code modified.

---

## Commit-by-Commit UI Impact Summary

- `b0304b7`: **High UI risk** due to CSP tightening with static hash on `script-src`.
- `4911e2c`: **Medium UI risk** due to broad component/runtime sync changes.
- `36ec636`: **No UI risk** (documentation only).

---

## Conclusion

The most probable trigger for the "UI gone / sections missing" behavior is the CSP `script-src` change in `apps/www/next.config.ts` from commit `b0304b7`.  
The component sync commit `4911e2c` is a secondary risk area, but evidence points first to script execution/hydration blocking caused by CSP.

## Recommended Guardrails

1. Keep CSP compatible with Next runtime (nonce-based or framework-safe approach), not a single static hash for all scripts.
2. When applying security fixes, validate UI routes (`/`, `/components`, `/docs`, `/templates`) with browser + console checks before commit.
3. Separate security header changes from component/refactor commits to isolate regressions faster.
